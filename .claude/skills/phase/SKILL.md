# Phase 실행 절차

인자가 `status`이거나 없으면: PLAN.md 진행 상태 체크박스를 읽고, 완료/미완료 phase와 다음에 할 phase를 보고하고 끝낸다.

인자가 숫자 N이면 아래 절차를 순서대로 수행한다:

## 1. 범위 확인

- `PLAN.md`의 Phase N 절을 읽는다. 거기 적힌 작업 항목이 범위의 전부다 — **다음 phase 기능을 미리 만들지 않는다.**
- 선행 phase(0..N-1)의 진행 상태를 확인한다. 미완료 DoD가 있으면 **구현을 시작하지 말고** 무엇이 막혀 있는지, 특히 Phase 0의 사람 작업(키 발급 등)이 원인인지 보고하고 중단한다.

## 2. 규약 로드

Phase N의 작업 영역에 해당하는 규약 문서를 먼저 읽는다 (AGENTS.md "구현 규약 문서" 표):
tourapi(API·프록시) / algorithm(추천 엔진) / sensors(센서) / pwa(manifest·SW·배포) + `docs/ui.md`(화면 작업 시) + `docs/zones.md`·`docs/curation.md`(데이터 모듈 작업 시).

## 3. 구현

- AGENTS.md 절대 원칙 6개 우선. 위반이 의심되는 요구가 PLAN에 있으면 구현하지 말고 보고.
- ui.md의 ⚠(미확정) 항목을 만나면 제안값대로 구현하되, 완료 보고에 "⚠ 항목 사용됨"을 명시.
- zones/curation 미확정 값은 임시값 + `// TODO(zones.md)` 주석으로 처리 (Phase 5부터는 확정값 필수).

## 4. DoD 검증

- Phase N의 DoD 각 항목을 실제로 검증한다: 테스트 실행, 키/금지 문자열 grep, dev 서버 확인 등. **"구현했으니 됐다"로 체크하지 않는다.**
- "사람 검증" 표시 항목은 자동 체크하지 말고, 완료 보고에 사용자가 할 체크리스트로 정리한다.

## 5. 마무리

- PLAN.md의 해당 DoD 체크박스와 진행 상태를 갱신한다 (사람 검증 항목은 미체크로 남긴다).
- Phase 1에서 명령어가 확정되면 AGENTS.md "명령어" 섹션도 갱신한다.
- git 저장소면 phase 단위로 커밋한다.
- 완료 보고: 구현 요약 / DoD 검증 결과(항목별 통과 근거) / 사람 확인 필요 항목 / 다음 phase 안내.
