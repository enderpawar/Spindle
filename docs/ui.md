# UI 명세 v0.3

> 자동 구현 시 AI가 임의 판단하지 않도록 화면·연출·카피 톤을 고정한다. ⚠ 표시는 **제안 상태 — 사용자 확정 필요** (확정되면 ⚠ 제거).
> 모바일 세로(390×844 기준) 우선, 데스크톱은 중앙 고정폭(최대 480px) 레이아웃.

- 공통 반응형 기준: 280~480px 화면 폭에서 의도하지 않은 가로 스크롤이나 잘린 카드·문구가 없어야 한다. 홈의 테마·추천 목록은 폭에 맞춰 열 수가 바뀌는 그리드를 사용하고, 긴 출발점·장소명은 컨테이너 안에서 말줄임 또는 최대 2줄로 제한한다.

## 디자인 톤

- ⚠ 컬러: 딥 네이비(#0F2540, 밤바다) 바탕 + 선셋 오렌지(#FF7A45) 포인트, 방위별 보조 색상 8종
- ⚠ 폰트: Pretendard (시스템 폴백)
- 무드: "게임"이되 유치하지 않게 — 나침반·항해 모티프. 소주병 이미지·일러스트는 사용하지 않는다 (확정)
- 카피 톤: 짧고 다정한 존댓말, 이모지 최소. 실패도 긍정 프레임 ("이 방향은 바다예요. 범위를 넓혔어요")

## 화면 흐름

```
S0 온보딩(첫 방문 1회) → S1 홈/스핀 ⇄ S2 출발점 선택(여행 모드)
                              │ 스핀 정지
                              ▼
                         S3 방위 연출 → S4 결과 카드 ┬→ S5 공유 카드
                              ▲______________│       └→ S6 방향 코스 (Phase 7)
                                             └────────── (다시 돌리기)
```

## S0 온보딩 (첫 방문 1회, 건너뛰기 가능)

- 2장: ① "휴대폰을 돌리면, 방향이 정해져요" (현장 모드 스핀 개념) ② "덜 알려진 부산을 만나요" (분산 서사 한 줄)
- 마지막에 모드 안내: 부산 현지면 현장 모드(위치·나침반 권한), 아니면 여행 모드
- 설정의 [사용법 다시 보기]는 이 슬라이드를 반복하지 않는다. 실제 홈 화면으로 이동한 뒤 출발점→스핀→테마→도장깨기 요소를 차례로 비추는 4단계 코치마크를 표시하며, 이전·다음·건너뛰기·완료를 제공한다.

## S1 홈 = 스핀 화면 (핵심 화면)

- 좌측 상단 브랜드 로고는 설치 아이콘에서 흰 배경을 제거한 파란 별·궤도 심볼을 사용해 홈 화면·PWA 아이덴티티를 통일한다.
- 중앙: **나침반 로즈(compass rose) 원판** — 8방위 눈금. 현장 모드에서는 기기 방위각을 따라 회전하고, 여행 모드에서는 자동 스핀
- 원판 중앙은 캐릭터 이미지 대신 북쪽을 가리키는 양면 나침반 바늘과 작은 브랜드 별 허브를 사용한다. 캐릭터가 방위 눈금의 기능성을 가리지 않게 한다.
- 상단: 모드 배지(현장/여행) + 출발점 이름 (탭 → S2)
- 원판 아래: **이동시간 다이얼** 바를 좌우로 드래그해 20분~하루를 **1분 단위로 부드럽게** 조정 (스냅 없음) — 기본 40분. 트랙은 앵커 눈금(20·30·40·60·90·120·180·240분·하루) 사이를 보간해 짧은 예산 구간이 넓고, 끝까지 밀면 하루. 요약에는 분위기 라벨(가볍게/반나절/느긋하게/하루)과 정확한 시간을 함께 보여준다. 키보드는 화살표 ±1분·PageUp/Down ±30분·Home/End로 동일 범위를 조정할 수 있어야 한다.
- 다이얼 아래(접힘): 테마 덱 선택 (기본 "테마 없이") — Phase 7 전까지 미노출
- 현장 모드 스핀: 손가락 드래그가 아니라 **휴대폰을 실제로 돌리는 속도**에 맞춰 원판이 회전한다. 기기 회전이 멈추고 방위값이 안정되면 실제 방위각을 잠가 8방위로 분류한다. 화면만 추가로 도는 가짜 관성은 사용하지 않는다.
- 여행 모드 스핀: 센서 없이 `돌리기` 버튼으로 자동 스핀한다. 버튼을 누른 시간이나 손가락 속도는 결과에 영향을 주지 않는다.
- 현장 모드 첫 진입: 권한 요청 버튼("내 위치에서 돌리기") — 자동 프롬프트 금지, 거부 시 여행 모드 전환 안내

## S2 출발점 선택 (여행 모드)

- 프리셋 카드 3개: 부산역 / 남포동 / 영도(흰여울 입구) — 탭 1번으로 완료
- 하단: "지도에서 고르기" (Phase 5) — 4개 구 범위 밖 선택 시 가장 가까운 권역 경계로 스냅 + 안내
- 선택 즉시 S1 복귀

## S3 방위 연출 (스핀 정지 직후, ~2초)

- 정지한 방위 부채꼴 하이라이트 + 방위명 + **8방위 큐레이션 메시지** 1줄
- 목적지 공개 전 카드는 관광 패스 형태로 구성하고, 사진 미리보기·별이·"탭해서 먼저 보기"·자동 공개 진행선을 표시한다. 큰 물음표만 둔 빈 카드는 사용하지 않는다.
- 공개 카드 위에는 별도의 "○쪽이에요" 배지를 반복하지 않는다. 추천 장소 사진을 카드 중앙에 미리 보여주고, 신난 별이는 사진 우측 하단에 겹쳐 배치하며 바로 위 말풍선에 "여기 어때요?"를 표시한다.
- 인접 확장 발생 시 메시지 하단에 사유 1줄
- 자동으로 S4 전환 (탭으로 즉시 스킵 가능)

### 8방위 메시지 (Phase 5에서 확정, 예시 톤)

동: "시작과 새벽의 방향. 떠오르는 해를 만나러 가요" — 나머지 7종 동일 톤·길이(공백 포함 25자 내외)로 작성

## S4 결과 카드

위에서 아래로:
1. 대표 이미지 (없으면 방위 색상 배경 + 나침반 아이콘 폴백) — 긴 화면에서는 높이가 화면의 ~38%까지 자라 빈 공간을 흡수한다. 배지·오버레이 칩은 올리지 않는다 (우상단 [사진] 제외). 이미지 탭 = 갤러리.
2. "숨은 명소" 배지 (T3만) / 축제 특별 카드는 이 위에 별도 카드 (Phase 7)
3. 관광지명 + 보조줄 "카테고리 · 구 · 도보 약 n분" + 주소 한 줄 — 설명보다 먼저. 이어서 핵심 소개 3~4줄 (detailCommon2 overview 앞 문장 최대 3개·160자 이내, 전체 원문은 [자세히 보기]). 운영·휴무는 프리뷰에 노출하지 않는다.
4. 주소·이동·운영·휴무 **원문 4행 표는 [자세히 보기] 시트**에서 설명보다 먼저 배치한다. TourAPI 원문의 `<br>` 표기는 실제 줄바꿈으로 정리한다.
5. 버튼: **[길찾기]** (카카오맵·네이버 선택) / [다른 후보] (나머지 2개 순환) / [다시 돌리기] (S1)
6. 코스 진입: **중립 톤 카드** "이 방향으로 코스 짜기 — 가까운 장소 2~4곳을 이어 코스로 만들어요" → S6. 방위색·강조색을 쓰지 않아 주 CTA(길찾기)와 시각 위계가 경쟁하지 않게 한다.
- 길찾기에서 새 도장을 얻으면 별도 토스트로 버튼을 가리지 않고, 길찾기 시트 안에 "○○구 여행 도장을 받았어요" 인라인 안내를 표시한다.

- 로딩: 스켈레톤 카드. 에러: "잠시 연결이 어려워요" + [재시도] (빈 화면 금지)

## S5 공유 카드

- 1080×1920(인스타 스토리) PNG 미리보기: 방위 색상 배경 + 관광지명 + 방위 메시지 + 이미지 + "Spindle이 정해준 오늘의 방향"
- 버튼: [공유하기] (Web Share API) / [저장] (다운로드 폴백)
- **"한국관광공사"·"KTO" 표기 절대 금지** — 출처 필요 시 "TourAPI" (절대 원칙 6)

## S6 방향 코스 (Phase 7)

- 상단: 확정 방위 + 출발점 + 이동시간 다이얼 요약. 스핀 결과를 바꾸는 별도 방위 선택은 제공하지 않는다.
- 본문: 방문 순서대로 2~4개 장소를 세로 목록으로 표시. 각 항목은 순번·대표 이미지·관광지명·이전 지점에서의 보정 이동시간·운영상태·[길찾기]를 포함한다.
- 하단: 전체 보정 이동시간과 "예상 이동시간이며 실제 경로와 다를 수 있어요" 안내, [다시 돌리기] 제공.
- 인접 방위 확장 또는 목표 장소 수 축소가 발생하면 목록 위에 사유를 한 줄로 표시한다.
- 후보가 2개 미만이면 S4를 유지하고 "이 방향에서는 코스를 만들 장소가 부족해요"를 표시한다.
- 로딩은 코스 목록 스켈레톤, 일부 상세 호출 실패는 해당 장소를 제외하고 재구성, 재구성 후 2개 미만이면 S4 폴백.

## 공통 규칙

- 획득 도장은 파란 선화의 "참 잘했어요" 별이 도장 이미지를 사용하며, 흰색·체크무늬 배경 없이 투명하게 표시한다.
- 로딩·에러·빈 상태 없는 화면 금지 (submission-check 항목)
- 터치 타겟 44px 이상, 세로 스크롤 1화면 이내 (S4·S6 제외)
- 모션은 기능 상태를 **토스식 스프링 감각으로 또렷하게** 전달하되(의미 없는 장식 모션은 지양), 화면 전환은 좌우 슬라이드 없이 **깊이(z축) 큐**로 위계를 표현한다. 탭 간 이동은 잔잔한 디졸브, 더 깊은 화면 진입(forward)은 새 화면이 살짝 확대되며 안착하는 zoom-in, 상위 복귀(back)는 살짝 축소되며 안착하는 zoom-out으로 처리한다(모두 가로 이동 없음). 스핀→공개→결과만 페이드·확대와 팝을 사용하며 하단 내비게이션은 고정한다.
- 버튼은 역할별 눌림 피드백을 제공한다. 누르는 순간은 70ms 이내로 빠르게 반응하고 Chrome의 빠른 클릭에서도 press 상태를 최소 110ms 보장한 뒤, 손을 떼면 약 440ms의 **또렷한 스프링 곡선(1.0을 약 6% 지나쳤다 정착)**으로 복원한다. 일반 버튼은 미세 축소, 주요 CTA·스핀 버튼은 더 깊은 축소·하강(scale 0.94)으로 쫀득하게, 카드형 버튼은 밝기와 깊이를 함께 조절한다. 시트·다이얼로그·상태 안내는 짧은 등장만 허용한다.
- **히어로 순간**(리빌 카드·별이 마스코트·결과 카드·도장 획득 안내·하단 탭 선택)은 등장 시 살짝 지나쳤다 정착하는 팝(spring-pop)으로 "즐거운 순간"을 만든다. 마스코트 팝인·도장 찍힘은 각 1회만 재생하고 무한 반복하지 않는다.
- 새 카드와 후보·테마·존 변경 결과는 스프링 팝의 상승·페이드로 나타내고, 카드 목록은 항목당 45ms 간격으로 최대 225ms까지만 순차 등장한다. 관광지 사진은 로드 완료 시 420ms 디졸브로 폴백 위에 표시한다.
- 지원 브라우저에서는 View Transition API로 이전·다음 화면을 320~460ms 동안 겹쳐 전환한다(이전 화면 고정 + 새 화면이 깊이 큐로 페이드인). 미지원 브라우저는 동일한 진입 모션으로 폴백한다.
- `prefers-reduced-motion: reduce`에서는 화면 이동·확대·팝·무한 반복과 코치마크 자동 스크롤을 즉시 전환으로 대체한다.
