# 방향 기반 여행 코스 명세 v0.1

> 상태: **Phase 7 구현 기준**. Phase 1~6의 단일 추천 MVP를 대체하지 않고 결과 카드에서 선택적으로 확장한다.
> 추천 점수와 존 이동비용은 각각 `.claude/skills/algorithm/SKILL.md`, `docs/zones.md`를 그대로 사용한다.

## 1. 목적과 진입점

- 사용자가 스핀으로 정한 방향을 유지하면서 실제로 이어서 방문할 수 있는 2~4개 장소를 제안한다.
- 진입점은 S4 단일 추천 결과의 **[이 방향으로 코스 짜기]**다. 코스 기능 때문에 기존 "다른 후보 보기"·"다시 돌리기" 흐름을 변경하지 않는다.
- 최초 추천 장소가 코스의 첫 장소이자 방향 앵커다. 코스를 만들 때 스핀 결과나 첫 장소를 다시 추첨하지 않는다.

## 2. 입력과 출력

### 입력

- 단말 내 출발점 좌표
- 확정 방위각과 8방위 부채꼴
- 이동시간 다이얼(가볍게/반나절/하루)
- 현재 단일 추천 POI
- 실시간 호출 후 세션 메모리에 있는 POI 후보와 상세정보
- `docs/zones.md` 존-교량 데이터와 `docs/curation.md` 티어 데이터

### 출력

| 다이얼 | 목표 장소 수 | 총 보정 이동시간 상한 |
|---|---:|---:|
| 가볍게 | 2곳 | 20분 |
| 반나절 | 3곳 | 40분 |
| 하루 | 4곳 | 권역 내 무제한 |

- 총 보정 이동시간은 `출발점 → 첫 장소 → ... → 마지막 장소`의 구간별 이동비용 합계다.
- 각 장소에는 방문 순서, 이전 지점으로부터의 보정 이동시간, 운영상태, 장소별 길찾기 딥링크를 표시한다.
- 시간은 존-교량 모델의 근사치이므로 정밀 경로 또는 도착시간으로 표현하지 않는다.

## 3. 후보 자격

후속 장소는 아래 조건을 모두 만족해야 한다.

1. 현재 코스에 같은 contentId가 없어야 한다.
2. 추천 점수 산식에서 운영 상태가 0이 아니어야 한다.
3. 원래 출발점에서 본 방위가 현재 코스의 기준 부채꼴에 속해야 한다.
4. 추가 후 총 보정 이동시간이 현재 다이얼 상한을 넘지 않아야 한다.
5. 세션 메모리에 존재하는 실시간 TourAPI 응답에서 얻은 후보여야 한다.

- 최초 추천이 인접 방위 확장으로 선정됐다면 해당 확장 부채꼴을 코스 기준 방위로 사용하고 기존 확장 사유를 유지한다.
- 후속 후보가 부족하면 확정 방위의 양쪽 인접 부채꼴까지 허용한다. 그래도 부족하면 장소 수를 줄이며, 코스만을 위해 전방위로 확장하지 않는다.

## 4. 구성과 방문 순서

1. 최초 추천 장소를 첫 장소로 고정한다.
2. 현재 위치(처음에는 첫 장소)에서 이동비용이 가장 낮은 적격 후보를 다음 장소로 선택한다.
3. 이동비용이 같으면 추천 점수가 높은 후보, 그마저 같으면 contentId 오름차순으로 선택해 결과를 재현 가능하게 한다.
4. 목표 장소 수에 도달하거나 더 이상 적격 후보가 없을 때 종료한다.

- 2곳 이상이면 목표 수보다 적어도 축소 코스를 제공하고 이유를 표시한다.
- 2곳 미만이면 코스를 생성하지 않고 S4 단일 추천 결과를 유지한다.
- 장소 상세 호출 하나가 실패하면 해당 후보를 제외하고 같은 규칙으로 한 번 재구성한다. 재구성 후 2곳 미만이면 S4로 폴백한다.

## 5. 개인정보·데이터 규칙

- 방향 판정, 후보 필터, 존 판정, 구간 이동비용, 방문 순서는 모두 단말 내에서 계산한다.
- 사용자 GPS 좌표·선택 출발점 좌표·방위각·코스 배열을 프록시나 외부 경로 API로 전송하지 않는다.
- POI 조회는 지역코드 기반 실시간 호출만 사용한다.
- POI 후보·상세 응답·생성 코스는 메모리/세션 범위에서만 유지하고 localStorage·IndexedDB·Cache Storage·파일·서버 DB에 저장하지 않는다.
- 지도 딥링크는 사용자가 장소별 [길찾기]를 누를 때만 연다. 앱 내부에서 다중 경유 경로 API를 호출하지 않는다.

## 6. 필수 테스트

- 첫 장소와 기준 방위가 코스 생성 전후 동일하다.
- 가볍게 20분, 반나절 40분 경계값에서 포함·제외가 정확하다.
- 같은 contentId가 중복되지 않는다.
- 같은 입력 후보 집합이면 방문 순서가 동일하다.
- 목표 수 미달 시 2곳 이상은 축소 코스, 2곳 미만은 단일 추천 폴백으로 동작한다.
- 바다 방향에서 인접 방위 확장 사유가 코스 화면에도 유지된다.
- 네트워크 요청에 사용자 좌표·방위각이 없고 영속 저장소에 API 응답·코스가 남지 않는다.
